== 📖 문서 개요

=== 시스템 소개
글나무 출석부 시스템은 모임 출석을 효율적으로 관리하기 위한 웹 기반 시스템입니다. +
사용자 권한에 따라 다양한 출석 관리 기능을 제공합니다.

=== 버전 정보
- **문서 버전**: v1.0.0
- **베이스 URL**: `아직 없음`

== 🔐 인증 및 권한

=== 인증 방식
- **OAuth 2.0**: 카카오 소셜 로그인
- **JWT Token**: 액세스 토큰 기반 인증
- **Refresh Token**: 쿠키 기반 토큰 갱신

// 추후 수정할 것
=== 권한 레벨
|===
| 권한 | 설명 | 접근 가능 기능

| PUBLIC | 모든 사용자 | 로그인, 토큰 재발급
| MEMBER | 인증된 사용자 | 개인 정보 조회/수정, 출석 처리, 로그아웃
| STAFF | 운영진 | 모임 생성/관리, 출석 현황 조회
| ADMIN | 관리자 | 모든 기능 + 회원 권한 관리, 이름 변경
|===

== ❌ 에러 코드

=== HTTP 상태 코드
|===
| 상태 코드 | 설명 | 발생 상황

| 200 | 성공 | 요청이 정상적으로 처리됨
| 400 | 잘못된 요청 | 요청 파라미터가 잘못되었거나 누락됨
| 401 | 인증 실패 | 토큰이 유효하지 않거나 만료됨
| 403 | 권한 없음 | 해당 기능에 대한 접근 권한이 없음
| 404 | 리소스 없음 | 요청한 자원이 존재하지 않음
| 417 | 유효성 검사 실패 | 입력값의 형식이나 제약조건에 맞지 않음
| 422 | 처리 불가능 | 요청은 올바르지만 현재 상태에서 처리할 수 없음
| 500 | 서버 오류 | 서버 내부에서 오류가 발생함
|===

=== 응답 구조
모든 API는 다음과 같은 공통 응답 구조를 가집니다:

```json
{
    "code": 200,
    "message": "Success",
    "data": { /* 실제 응답 데이터 */ }
}
```

== 🚀 시작하기

=== 1단계: 인증
1. **카카오 로그인**: link:login-api-guide.html#카카오-oauth-로그인[GET /login/oauth/kakao] 호출
2. **토큰 저장**: 응답에서 받은 액세스 토큰을 Authorization 헤더에 포함
3. **토큰 갱신**: 토큰 만료 시 link:login-api-guide.html#액세스-토큰-재발급[POST /login/re-issue/accessToken] 호출

=== 2단계: 기본 정보 입력
1. **정보 입력 여부 확인**: link:member-api-guide.html#모임원-정보-입력-여부-확인[GET /member/info]
2. **정보 미입력 시**: link:member-api-guide.html#모임원-정보-수정[PATCH /member/info]로 개인정보 등록

=== 페이지네이션
목록 조회 API는 다음 파라미터를 지원합니다:
- `page`: 페이지 번호 (기본값: 0)
- `size`: 페이지 크기 (기본값: 20, 최대: 100)
- `sort`: 정렬 조건 (예: createdAt,desc)

=== 에러 처리 예시
```javascript
try {
  const response = await fetch('/api/member/info', {
    headers: { 'Authorization': `Bearer ${token}` }
  });

  const result = await response.json();

  if (result.code !== 200) {
    throw new Error(result.message);
  }

  return result.data;
} catch (error) {
  console.error('API 오류:', error.message);
}
```
