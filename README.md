# 글나무 출석부 시스템

**글나무 출석부 시스템**은 모임 출석을 효율적으로 관리하기 위한 웹 기반 시스템입니다.  
운영진, 관리자, 일반 사용자 등 사용자 권한에 따라 다양한 출석 관리 기능을 제공합니다.

---

## 🛠️ 주요 기능

### 📌 공통 기능 (일반 사용자)
- 로그인 (U)
- 모임별 개인 출석 이력 확인 (R)
- 출석 처리 (U)
- (추후 개발 기능) 토론 참여 알림 (R)
- (추후 개발 기능) 모임별 본인 토론 조 확인 (R)
- (추후 개발 기능) 오늘의 발제 작성 (C)
- (추후 개발 기능) 모임별 조별 발제 조회 (R)

### 👑 모임장·부모임장·관리자
- 회원 등급 수정 (U)

### 🛡 운영진·준운영진
- 모임 생성 (C)
- 모임 목록 조회 (R)
- 모임 출석용 QR 생성 및 조회 (R)
- 모임별 지각자 조회 (R)
- (추후 개발 기능) 모임별 조생성 기능 (모임 생성자만 가능 - 자동, 수동 가능) (C)
- (추후 개발 기능) 조별 오늘의 발제 조회 (R)

---

## 🖥️ 주요 화면 구성

- **홈 화면**
  - 로그인 버튼
- **출석용 QR 화면**
- **개인 정보 조회 화면**
  - 모임별 출석 이력 확인
  - 모임별 본인 토론 조 확인
- **운영진 전용 화면**
  - 모임 생성 버튼
  - 생성된 모임 목록 및 지각자 조회
  - 모임별 조 생성
  - 조별 오늘의 발제 조회
  - 모임별 오늘의 발제 조회
- **관리자 전용 화면**
  - 회원 등급 수정
- **기타**
  - 토론 참여 알림 팝업 (모임별 본인 토론 조 확인 화면 이동 버튼)

## ✏️ 커밋 컨벤션

프로젝트에서는 다음과 같은 커밋 메시지 규칙을 사용합니다.

### ✅ 커밋 타입

| 타입 | 설명 |
|------|------|
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `docs` | 문서 수정 (README 등) |
| `refactor` | 리팩토링 (기능 변화 없이 코드 개선) |
| `test` | 테스트 코드 추가 또는 수정 |
| `chore` | 설정 파일 수정 등 기타 작업 |

### ✅ 커밋 메시지 예시

```bash
feat: 로그인 기능 추가
fix: 출석 이력 조회 오류 수정
docs: README에 커밋 전략 추가
```

---

## 🌿 브랜치 전략

- `prod` ← 운영용 메인 브랜치  
- `dev` ← 개발용 브랜치

> 모든 개발 작업은 `dev` 브랜치에서 수행 후, 검토를 거쳐 `prod` 브랜치로 병합합니다.
