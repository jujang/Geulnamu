Index 페이지로 이동

POST

모임 생성

  • 접근 가능 권한 레벨: STAFF / 로그 생성

  • 운영진급 모임원이 모임을 생성하는 용도의 API

  • 모임의 개최 일정은 미래 시간만 가능함.

  • 지각 기준 시간을 설정할 수 있는데, 설정하지 않는다면 모임 개최 시간 자체가 지각 기준 시간이 됨. (지각 기준 시간은 모임 개최 시간 보다 빠르면 안 되고, 같은 날 안에서만 설정이 가능함.)

request

POST /meetings/create HTTP/2.0
Content-Type: application/json;charset=UTF-8
Authorization: Bearer access_token
Accept: application/json
Content-Length: 276
Host: docs.api.com

{
  "meetingType" : "REGULAR",
  "meetingName" : "제 200회 정기모임",
  "meetingDate" : "21260614 10:30",
  "lateThresholdTime" : "21260614 10:45",
  "meetingPlace" : "추후 공지 예정 (합정역 주변 카페)",
  "description" : "늦지 않게 오세요~"
}

Authorization

액세스 토큰

request-fields

필드명 타입 필수여부 제약조건 설명

meetingType

String

true

'REGULAR', 'FLASH', 'SPECIAL' 중 하나의 값

모임 종류

meetingName

String

true

한글, 영문, 숫자, 공백 및 일부 특수분자(: / [ ] ( ) ~ _ -)만으로 사용한 1자 이상, 70자 이하

모임 제목

meetingDate

String

true

yyyyMMdd HH:mm 형식으로 이뤄진 미래 시간대의 문자열

모임 개최일자

lateThresholdTime

String

false

yyyyMMdd HH:mm 형식으로 이뤄진 미래 시간대의 문자열

지각 기준 시간(모임 개최일자보다 빠르면 안 됨. 입력 안 할 시 모임 개최시간과 동일값 처리)

meetingPlace

String

true

한글, 영문, 숫자, 공백 및 일부 특수분자(: / [ ] ( ) ~ _ -)만으로 사용한 1자 이상, 255자 이하

모임 장소

description

String

false

형식, 길이 제한 없는 문자열

PATCH

모임 수정 - 기본 정보

  • 접근 가능 권한 레벨: STAFF / 로그 생성

  • 운영진 모임원이 모임을 수정하는 용도의 API (운영진 등급 중에서는 개설 당사자만 가능)

  • 토론 시작 전, 미래 시간으로만 수정 가능 (관리자 급은 토론 시작 후에도 수정 가능)

request

PATCH /meetings/1/discussion HTTP/2.0
Content-Type: application/json;charset=UTF-8
Authorization: Bearer access_token
Accept: application/json
Content-Length: 93
Host: docs.api.com

{
  "discussionTime" : "20260501 12:30",
  "alarmMessage" : "모두 올라와주세요~"
}
