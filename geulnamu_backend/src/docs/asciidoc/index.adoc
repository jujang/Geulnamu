== 📖 문서 개요

=== 시스템 소개
글나무 출석부 시스템은 모임 출석을 효율적으로 관리하기 위한 웹 기반 시스템입니다. +
사용자 권한에 따라 다양한 출석 관리 기능을 제공하며, 토론 그룹 관리, 발제 작성, 액션 히스토리 추적 등의 기능을 포함합니다.

=== 버전 정보
- **문서 버전**: v0.3.0
- **베이스 URL**: `https://api.geulnamu.com` (개발 중)
- **API 버전**: v1

=== 핵심 기능
- **출석 관리**: QR 코드/GPS 기반 출석 처리 및 이력 관리
- **토론 그룹**: 자동/수동 그룹 편성 및 관리
- **발제 시스템**: 조별 발제 작성 및 승인 프로세스
- **액션 로깅**: 시스템 사용 이력 자동 추적
- **VoC 관리**: 사용자 의견 및 문의사항 처리

== 🔐 인증 및 권한

=== 인증 방식
- **OAuth 2.0**: 카카오 소셜 로그인
- **JWT Token**: 액세스 토큰 기반 인증
- **Refresh Token**: 쿠키 기반 토큰 갱신
- **보안**: HTTPS 통신 및 토큰 암호화

=== 권한 레벨
|===
| 권한 | 설명 | 접근 가능 기능

| **PUBLIC** | 모든 사용자 | 로그인, 토큰 재발급
| **MEMBER** | 인증된 사용자 | 모든 사용자 기능 + 개인 정보 조회/수정, 출석 처리, 발제 작성, 로그아웃
| **STAFF** | 운영진/준운영진 | 인증된 사용자 기능 + 모임 생성/관리, 출석 현황 조회, 토론 그룹 편성
| **ADMIN** | 모임장/부모임장/관리자 | 운영진 기능 + 회원 권한 관리, 모임원 이름 변경, 시스템 관리
|===

== ❌ 에러 코드

=== HTTP 상태 코드
|===
| 상태 코드 | 설명 | 발생 상황

| **200** | 성공 | 요청이 정상적으로 처리됨
| **400** | 잘못된 요청 | 요청 파라미터가 잘못되었거나 누락됨, 그 외 분류되지 않은 오류
| **401** | 인증 실패 | 토큰이 유효하지 않거나 만료됨
| **403** | 권한 없음 | 해당 기능에 대한 접근 권한이 없음
| **404** | 리소스 없음 | 요청한 자원이 존재하지 않음
| **417** | 유효성 검사 실패 | 입력값의 형식이나 제약조건에 맞지 않음
| **422** | 처리 불가능 | 요청은 올바르지만 현재 상태에서 처리할 수 없음
| **460** | 비활성화 계정 | 비활성화 계정으로 요청을 보냈기에 접근 권한이 없음
| **500** | 서버 오류 | 서버 내부에서 오류가 발생함
|===

== 🚀 시작하기

=== 1단계: 인증
1. **카카오 로그인**: link:login-api-guide.html#카카오-oauth-로그인[GET /login/oauth/kakao] 호출
2. **토큰 저장**: 응답에서 받은 액세스 토큰을 Authorization 헤더에 포함
3. **토큰 갱신**: 토큰 만료 시 link:login-api-guide.html#액세스-토큰-재발급[POST /login/re-issue/accessToken] 호출

=== 2단계: 기본 정보 입력
1. **정보 입력 여부 확인**: link:member-api-guide.html#모임원-정보-입력-여부-확인[GET /member/info]
2. **정보 미입력 시**: link:member-api-guide.html#모임원-정보-수정[PATCH /member/info]로 개인정보 등록

=== 3단계: 기능 이용
- **출석**: QR 코드 스캔을 통한 출석 처리
- **이력 조회**: 개인 출석 현황 및 참여 이력 확인
- **토론 그룹**: 배정된 토론 그룹 확인 및 발제 작성
- **의견 제시**: VoC를 통한 건의사항 등록

== 🔧 개발자 참고사항

=== 요청 헤더
```http
Content-Type: application/json
Authorization: Bearer {access_token}
Accept: application/json
User-Agent: GeulnamuApp/1.0
```

=== 페이지네이션
- 특정 API 들은 다음 파라미터를 지원합니다
- `page`: 페이지 번호 (예: 1)
- `size`: 페이지 크기 (예: 10)
- `sort`: 정렬 조건 (예: id, createdAt)
- `isAsc`: 오름차순 여부 (true, false)

=== 날짜 포맷
- **ISO 8601 형식**: `2024-12-31T23:59:59Z`
- **로컬 날짜**: `2024-12-31`
- **시간대**: KST (UTC+9)

=== 에러 처리 예시
```javascript
try {
  const response = await fetch('/api/member/info', {
    headers: { 
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json'
    }
  });

  const result = await response.json();

  if (result.code !== 200) {
    throw new Error(result.message);
  }

  return result.data;
} catch (error) {
  console.error('API 오류:', error.message);
  // 에러 처리 로직
}
```

=== 보안 고려사항
- **토큰 보안**: 액세스 토큰은 메모리에만 저장, 리프레시 토큰은 HttpOnly 쿠키
- **HTTPS 필수**: 모든 API 통신은 HTTPS를 통해서만 허용
- **입력 검증**: 모든 사용자 입력에 대한 서버 측 검증 수행
- **액션 로깅**: 중요한 작업은 자동으로 기록되어 감사 추적 가능
